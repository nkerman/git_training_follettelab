# git_training_follettelab
Repository, or "repo" used for training students to use Git Version Control and GitHub.

Welcome! We're going to go through a few training tasks to get the hang of Git. We'll also practice our **markdown** `skills` _while we're at it_!

## Demo section
We'll start with Nat reviewing the fundamental git commands: `clone`, `add`, `commit`, `push`, `merge`.

## Your turn!

Students are all going to work in pairs, and hopefully figure out how to do things together. If you get stuck though, I'll be around for questions!

### Setup the repo
1. If you don't already have one, go ahead and make a GitHub account.
2. Start out by `forking` this repo to your own GitHub account. This makes your own copy which you can control, make edits to, and then connect your changes back to the original repository or "repo".
3. `Clone` the forked repo to your computer (_Advanced version: clone this repo, and **then** add your fork of it as a different remote._) Your `upstream` branch should be your own fork.
### Explorate the repo
1. Explore the repo locally, see what's in it! Use the CLI and the GUI/"finder" app.
2. Find and run the python file in the repo. Does it make any output files? Does git "see" those files? Figure out why/why not!
### Adding your name
1. Make a new branch called `<your-name>` (e.g., mine would be `nat`).
2. Add your name to the list of students, then `add`, `commit, and`` your changes.
3. Create a merge request to the original repo's `main` branch. We'll probaby (*hopefully!*) run into some merge conflicts and work through them together.
4. Switch back to the main branch on your local clone of the repo, and pull your merged changes from the `upstream` (original) repo.
### Improving the plotter code
1. Make a new branch with a descriptive name, then make some changes to `plotter.py` to make it more interesting! This could be anything from changing the color to adding random errors, etc.
2. Go through the same process you used to add your name to merge these changes into the official repo.
